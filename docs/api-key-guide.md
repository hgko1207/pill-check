# 식약처 OpenAPI 키 발급 가이드

PillCheck V1은 **활용신청 4건**, **결과적으로 인증키 2개**를 사용합니다 (모두 무료).

## 신청 vs 키 — 헷갈리지 마세요

| 발급처 | 활용신청 | 인증키 | 환경 변수 |
|---|---|---|---|
| **공공데이터포털 (data.go.kr)** | ① 의약품 제품 허가정보<br>② **DUR 성분정보** (앱의 핵심)<br>③ DUR 품목정보 (보조) | **1개** (회원당 인증키 하나로 세 API 모두 호출) | `VITE_NEDRUG_API_KEY` |
| **식품안전나라** (별도 포털) | ④ 건강기능식품 정보 | **1개** (별도 회원가입·별도 키) | `VITE_FOODSAFETY_API_KEY` |
| **합계** | **4건 활용신청** | **2개 인증키** | 2개 |

> 💡 data.go.kr은 회원당 인증키 1개로 여러 API를 호출하는 구조입니다. 활용신청은 "이 API 쓸 권한 등록"이고, 키 발급은 회원가입 시 자동(또는 첫 활용신청 시). 그래서 의약품 허가정보 + DUR 두 종류를 따로 신청해도 키는 같습니다.

| 데이터셋 (정확한 이름) | 사용 시점 | 우선순위 |
|---|---|---|
| 식품의약품안전처_의약품 제품 허가정보 | Week 0~1 (검증 + 약 검색·등록) | 필수 |
| 식품의약품안전처_**의약품안전사용서비스(DUR)성분정보** | Week 0~3 (성분 매칭 — 와파린+비타민K 같은 케이스) | **필수 (앱의 핵심)** |
| 식품의약품안전처_의약품안전사용서비스(DUR)품목정보 | Week 2~3 (처방약 ↔ 처방약 품목 매칭) | 권장 (보조) |
| 식품안전나라 건강기능식품 정보 | Week 2~ (영양제 마스터) | 필수 (식품안전나라 별도 키) |

> ⚠️ **왜 성분정보가 필수**: PillCheck의 wedge는 "처방약 + 영양제 상호작용"입니다. 영양제(건강기능식품)는 식약처 의약품 코드가 없어 **품목 단위 매칭이 불가능**합니다. 반드시 **성분 단위**로 매칭해야 합니다 (와파린 성분 ↔ 비타민K 성분 등). `scripts/week0/validate_dur.py`도 DUR 성분정보 엔드포인트(`DURIrdntInfoService03`)를 호출합니다.

> 📝 **혼동 주의 — 활용신청 안 해도 되는 것**: data.go.kr에서 "DUR"로 검색하면 위쪽에 `한국의약품안전관리원_연령금기`, `_임부금기약물`, `_노인주의약물` 같은 **CSV 파일** 데이터셋이 나옵니다. 이건 **단일 약물 주의사항** (특정 약이 노인에게 주의 등)이고, **병용 상호작용**과는 다릅니다. PillCheck V1에 안 씁니다.

**소요 시간:** 신청 자체는 15분. 승인은 자동 즉시 ~ 5영업일.

## 1️⃣ 공공데이터포털 (data.go.kr) — 1번/2번 한 번에

### 1-1. 회원가입

1. https://www.data.go.kr 접속
2. 우측 상단 "회원가입" 클릭
3. **일반회원** 선택
4. 이메일/휴대폰 인증 (휴대폰 본인인증 1회 필요)
5. 가입 완료 → 로그인

### 1-2. 첫 번째 활용신청 — "의약품 제품 허가정보"

1. 메인 검색창에 **`의약품 제품 허가정보`** 입력
2. 결과에서 **"식품의약품안전처_의약품 제품 허가정보"** 클릭
   - 제공기관: 식품의약품안전처
   - 분류체계: 보건의료
3. 페이지 우측 또는 상단의 **"활용신청"** 버튼 클릭
4. 활용신청 폼 작성:

| 항목 | 입력값 |
|---|---|
| 활용목적 선택 | "앱개발(웹·모바일)" 또는 "기타" |
| 시스템유형 | **일반** |
| 활용목적 | `개인 학습 및 가족용 영양제·일반의약품 상호작용 체크 PWA 프로토타입 개발. 식약처 의약품 마스터 데이터를 사용하여 부모님이 복용 중인 약과 신규 구매 영양제의 상호작용을 확인하는 도구.` |
| 라이선스 표시 동의 | ✅ 체크 |

5. **신청** 클릭 → 보통 **자동승인** (1분 ~ 1시간)

### 1-3. 두 번째 활용신청 — **"DUR 성분정보"** (앱의 핵심)

1. 메인 검색창에 **`DUR`** 입력 (또는 `의약품안전사용서비스`)
2. 검색 결과 하단의 **"오픈 API"** 섹션에서 **"식품의약품안전처_의약품안전사용서비스(DUR)성분정보"** 클릭
   - 제공기관: 식품의약품안전처
   - "성분정보" 라는 단어 확인 (품목정보 아님)
3. **활용신청** → 1-2와 동일한 활용목적 사용 가능 → 신청
4. 자동승인 대기

> ⚠️ **품목정보가 아닌 성분정보입니다.** 품목정보는 다음 단계(1-4)에서 보조로 신청. 성분정보가 PillCheck wedge의 핵심.

### 1-4. 세 번째 활용신청 — "DUR 품목정보" (보조)

1. 같은 검색 결과에서 **"식품의약품안전처_의약품안전사용서비스(DUR)품목정보"** 클릭
2. **활용신청** → 신청

> 💡 품목정보는 처방약 ↔ 처방약 매칭 시 보조로 쓰입니다. V1 출시 후 데이터 보강 단계에 활용. 지금 같이 신청해두면 추후 또 1~5일 대기할 필요 없음.

### 1-5. 인증키 확인 (셋 다 승인된 후)

1. 우측 상단 **마이페이지** 클릭
2. 좌측 메뉴 **"오픈API"** → **"개발계정"**
3. 신청한 세 API가 목록에 보임 (의약품 제품 허가정보 / DUR 성분정보 / DUR 품목정보). 상태가 모두 **"승인"** 인지 확인.
4. 아무 API 이름 클릭 → 상세 페이지 진입
5. 상단의 **"일반 인증키 (Encoding)"** 옆 키를 **그대로 복사** (예: `kJ1aB%2BcD3eFgH4iJkLmNopq...`)

> ⚠️ **중요:** 두 가지 키가 있습니다 — `Encoding`과 `Decoding`. **반드시 `Encoding` 키를 복사하세요.** PillCheck 코드는 Encoding 키 기준으로 작성됨 (`%`, `+` 등이 그대로 남아있음).

> 💡 **세 API 모두 같은 인증키 하나로 작동합니다.** 어느 API 상세 페이지에서 키를 복사해도 같은 값. 의약품 제품 허가정보, DUR 성분정보, DUR 품목정보 모두 이 키 하나로 호출 가능.

### 1-6. 키 동작 sanity check (1분)

브라우저 주소창에 직접 붙여넣기 (`{KEY}` 자리에 본인 Encoding 키 그대로):

```
http://apis.data.go.kr/1471000/DrugPrdtPrmsnInfoService06/getDrugPrdtPrmsnDtlInq05?serviceKey={KEY}&type=json&item_name=타이레놀&numOfRows=1
```

**기대 결과:** JSON이 뜨고 타이레놀 제품 1건 보임.

**문제별 진단:**

| 보이는 것 | 원인 | 대응 |
|---|---|---|
| JSON + 타이레놀 정보 | ✅ 정상 | 다음 단계 |
| `SERVICE_KEY_IS_NOT_REGISTERED_ERROR` | 키 활성화 안 됨 | 5분~1시간 대기 후 재시도 |
| `LIMITED_NUMBER_OF_SERVICE_REQUESTS_EXCEEDS_ERROR` | 일일 한도 초과 | 다음 날 재시도. 개발 단계엔 잘 안 발생. |
| XML 응답 (JSON 아님) | `&type=json` 누락 또는 미지원 | URL 다시 확인. 그래도 XML이면 식약처 API가 일시 변경된 것 — 활용신청 시 받은 안내문서 참조. |
| HTTP 에러 페이지 | URL 오타 | 위 URL 그대로 복사 |

## 2️⃣ 식품안전나라 (건강기능식품) — 별도 포털

> ⚠️ data.go.kr과 **다른 사이트**입니다. 회원가입도 별도.

### 2-1. 회원가입

1. https://openapi.foodsafetykorea.go.kr 접속
2. 우측 상단 **"회원가입"** 클릭 (또는 로그인 → 회원가입)
3. 이메일 인증
4. 가입 완료 → 로그인

### 2-2. 인증키 발급 신청

1. 상단 메뉴 **"마이페이지"** 또는 **"인증키 발급"** 클릭
2. 신청 양식 작성:

| 항목 | 입력값 |
|---|---|
| 사용 사이트 URL | `http://localhost:5173` (또는 추후 배포할 도메인) |
| 사용 목적 | `개인 학습 및 가족용 영양제 비교 PWA 개발. 건강기능식품 마스터 데이터를 사용하여 사용자가 구매하려는 영양제의 성분을 확인하는 용도.` |

3. **신청** → 보통 **즉시 또는 1영업일 내 발급**

### 2-3. 인증키 확인

1. 마이페이지 → 발급된 인증키 표시 영역
2. 키를 **그대로 복사** (식품안전나라는 보통 인코딩 차이 없는 단순 alphanumeric)

## 3️⃣ 발급된 키 PillCheck에 적용

**총 2개의 키**를 두 군데(Python용·웹용)에 같은 값으로 적습니다.

| 변수명 | 값 | 어디서 발급 |
|---|---|---|
| `NEDRUG_API_KEY` (Python) / `VITE_NEDRUG_API_KEY` (웹) | data.go.kr Encoding 키 | data.go.kr 마이페이지 (의약품 허가정보 + DUR 둘 다 이 키 하나로 동작) |
| `FOODSAFETY_API_KEY` (Python) / `VITE_FOODSAFETY_API_KEY` (웹) | 식품안전나라 인증키 | openapi.foodsafetykorea.go.kr 마이페이지 |

### 3-1. Python 검증 스크립트용 (Week 0 GATE 1)

```powershell
cd d:\project\2026\pill-check\scripts\week0
copy .env.example .env
notepad .env
```

`.env` 파일에:
```
# data.go.kr 키 — 의약품 허가정보 + DUR 둘 다 이거 하나로 사용
NEDRUG_API_KEY=여기에_data.go.kr_Encoding_키_붙여넣기

# 식품안전나라 키 — 건강기능식품용 (Week 2부터)
FOODSAFETY_API_KEY=여기에_식품안전나라_키_붙여넣기
```

### 3-2. PWA 웹앱용 (Week 1~)

```powershell
cd d:\project\2026\pill-check\web
copy .env.example .env
notepad .env
```

`.env` 파일에 (값은 위 Python용과 **같은 키 그대로**):
```
# data.go.kr 키 — 의약품 허가정보 + DUR 둘 다 이거 하나로 사용
VITE_NEDRUG_API_KEY=여기에_data.go.kr_Encoding_키_붙여넣기

# 식품안전나라 키 — 건강기능식품용 (Week 2부터)
VITE_FOODSAFETY_API_KEY=여기에_식품안전나라_키_붙여넣기
```

> 💡 Python의 `NEDRUG_API_KEY`와 웹의 `VITE_NEDRUG_API_KEY`는 **완전히 같은 값**입니다. 단지 Vite는 클라이언트 노출용 환경변수에 `VITE_` 접두사를 요구해서 이름만 다를 뿐. 키 자체는 하나만 발급받아 두 곳에 같이 붙여넣으면 됩니다.

> 두 .env 파일은 모두 `.gitignore`에 의해 git에 안 올라갑니다. **GitHub에 노출 위험 없음.**

## 흔한 실수 체크리스트

- [ ] data.go.kr과 식품안전나라를 같은 사이트로 착각 (다른 회원가입)
- [ ] **Decoding 키**를 사용 (Encoding 키여야 함)
- [ ] 활용목적을 한 단어로 작성 ("테스트") → 가끔 반려. 한 줄 이상 구체적으로
- [ ] 활용신청 즉시 호출 → 인증키 활성화에 5분~1시간 걸릴 수 있음
- [ ] 키 신청은 했는데 마이페이지에서 키 확인 안 함 → 2번 신청 모두 승인됐는지 확인 필수
- [ ] `.env` 파일을 `.env.example`로 만든 뒤 git에 커밋 → **금물.** 항상 `.env`로 (gitignore 적용됨)

## 키가 안 와요 / 거절됐어요

- **3영업일 지나도 메일 없으면**: data.go.kr 마이페이지에서 신청 상태 확인. "심사중"이면 더 대기, "반려"면 사유 확인 후 재신청.
- **반려 사유 흔한 것**: "활용목적이 구체적이지 않음" → 위 가이드의 활용목적 문장 그대로 다시 사용
- **그래도 안 되면**: data.go.kr 1:1 문의 (응답 1~3일)
